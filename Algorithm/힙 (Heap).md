# 힙


## Python에서의 heap 명령어 : `heapq`모듈
Python 의 **`heapq`** 모듈은 일반적인 리스트를 **최소 힙(Min Heap)** 처럼 다룰 수 있게 해주는 기능을 제공합니다. 이 모듈은 별도의 객체를 생성하는 것이 아니라, 기존 리스트를 인자로 받아 힙의 속성(부모 노드의 값이 자식보다 작거나 같은 상태)을 유지하도록 돕습니다.

---

### 1. 주요 함수 설명

| 함수명 | 동작 설명 | 시간 복잡도 |
| --- | --- | --- |
| **`heapq.heappush(heap, item)`** | 리스트 `heap` 에 `item` 을 추가하면서 힙 속성을 유지합니다. | $O(\log n)$ |
| **`heapq.heappop(heap)`** | 힙에서 **가장 작은 값**을 삭제하고 반환합니다. 비어있으면 `IndexError` 가 발생합니다. | $O(\log n)$ |
| **`heapq.heapify(x)`** | 리스트 `x` 를 선형 시간 내에 제자리(In-place)에서 힙으로 변환합니다. | $O(n)$ |
| **`heapq.heappushpop(heap, item)`** | `item` 을 푸시한 후, `heappop()` 을 호출합니다. 각각 따로 쓰는 것보다 효율적입니다. | $O(\log n)$ |
| **`heapq.heapreplace(heap, item)`** | `heappop()` 을 먼저 실행하여 가장 작은 값을 반환하고, 그 후 `item` 을 푸시합니다. | $O(\log n)$ |

---

### 2. 고급 기능: 데이터 추출

단순히 최솟값을 빼는 것 외에, 상위 $k$ 개의 요소를 가져오는 효율적인 함수도 제공합니다.

* **`heapq.nlargest(n, iterable, key=None)`**: 데이터 집합에서 가장 큰 값 $n$ 개를 리스트로 반환합니다.
* **`heapq.nsmallest(n, iterable, key=None)`**: 데이터 집합에서 가장 작은 값 $n$ 개를 리스트로 반환합니다.

> **객관적 평가:** 단순히 정렬이 필요하다면 `sorted(iterable)[:n]` 이 낫지만, $n$ 이 전체 데이터 크기에 비해 매우 작을 때는 이 함수들이 메모리와 시간 측면에서 더 효율적입니다.

---

### 3. ⚠️ 주의사항 및 지적 사항

1. **최소 힙 고정:** Python 의 `heapq` 는 **최소 힙**만 지원합니다. 최대 힙(Max Heap)이 필요하다면 값에 마이너스(`-`) 부호를 붙여 저장하는 방식을 직접 구현해야 합니다.
2. **정렬 상태 오해:** `heapify()` 를 거친 리스트를 출력해 보면 `heap[0]` 은 항상 최솟값이지만, **나머지 요소들이 전체 정렬된 상태는 아닙니다.** 힙은 부모-자식 간의 관계만 보장할 뿐, 형제 노드 간의 순서는 보장하지 않기 때문입니다.
3. **안정성:** 동일한 우선순위를 가진 요소들의 순서는 보장되지 않으므로, 정교한 우선순위 큐가 필요하다면 `(priority, task_id, item)` 과 같은 튜플 형태로 데이터를 관리하는 설계가 필요합니다.

### 4. 사용 예시

```python
import heapq

numbers = [3, 1, 4, 1, 5, 9, 2]
heapq.heapify(numbers)  # 기존 리스트를 힙 구조로 변경

heapq.heappush(numbers, 0)  # 0 추가
print(heapq.heappop(numbers))  # 가장 작은 값인 0 출력

```

혹시 **최대 힙**을 구현하는 구체적인 테크닉이나, **우선순위 큐(`PriorityQueue`)** 클래스와의 성능 차이가 궁금하신가요? 구체적인 활용 사례를 말씀해 주시면 더 자세히 설명해 드릴 수 있습니다.
