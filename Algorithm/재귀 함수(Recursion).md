# 재귀 함수 (Recursion)

* **날짜:** 2026-03-09
* **카테고리:** #Algorithm #Programming_Basics #Python #Recursion

## 1. 개념 및 목적

재귀 함수는 정의 단계에서 **자기 자신을 재참조**하는 함수를 의미합니다. 큰 문제를 동일한 구조를 가진 더 작은 문제로 나누어 해결하는 '분할 정복' 논리를 바탕으로 하며, 코드를 직관적이고 간결하게 작성하는 것이 목적입니다.

## 2. 상세 작동 원리

### 2.1. 필수 구성 요소

재귀 함수가 무한 루프에 빠지지 않고 정상적으로 동작하려면 반드시 다음 두 가지가 포함되어야 합니다.

1. **기본 케이스 (Base Case):** 재귀 호출을 멈추는 종료 조건입니다. 이 조건이 없으면 함수는 자기 자신을 무한히 호출하다가 스택 오버플로(Stack Overflow) 에러를 발생시킵니다.
2. **재귀 단계 (Recursive Step):** 문제를 더 작은 단위로 쪼개어 자기 자신을 호출하는 과정입니다. 호출이 반복될수록 결국 기본 케이스에 도달하도록 설계해야 합니다.

### 2.2. 함수의 호출 스택 (Call Stack)

함수가 호출될 때마다 컴퓨터 메모리의 **스택(Stack)** 영역에는 함수의 실행 정보가 쌓입니다. 재귀 호출이 일어나면 기존 함수의 실행이 멈춘 채 새로운 함수가 스택 위로 쌓이며, 가장 나중에 호출된 함수가 종료되어야 비로소 이전 함수의 실행이 재개됩니다.

## 3. 예시 코드 (팩토리얼 계산)

재귀 함수의 가장 전형적인 예시인 팩토리얼($n!$) 구현 코드입니다.

```python
def factorial(n):
    # 1. 기본 케이스 (Base Case)
    if n <= 1:
        return 1
    
    # 2. 재귀 단계 (Recursive Step)
    # n! = n * (n-1)! 임을 이용합니다.
    return n * factorial(n - 1)

# 실행 결과 확인
result = factorial(5)
print(f"5 팩토리얼 결과: {result}")

"""
예상 출력:
5 팩토리얼 결과: 120
"""

```

def recur(num):

## 4. 기술적 분석

* **장점:** 알고리즘의 원리가 재귀적인 경우(예: 트리 순회, 하노이의 탑), 반복문보다 훨씬 가독성이 높고 수학적으로 명확한 코드를 작성할 수 있습니다.
* **단점:** 함수 호출마다 스택 메모리를 사용하므로 반복문($for$, $while$)에 비해 메모리 소비가 크고 속도가 상대적으로 느릴 수 있습니다.
* **파이썬의 특징:** 파이썬은 기본적으로 재귀 깊이를 1,000회 정도로 제한하고 있습니다. 매우 깊은 재귀가 필요하다면 `sys.setrecursionlimit()`을 통해 제한을 늘리거나 반복문으로 변환(Iterative approach)하는 것이 권장됩니다.
