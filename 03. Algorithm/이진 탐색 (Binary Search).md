# 이진 탐색 (Binary Search)

* **날짜:** 2026-03-11
* **카테고리:** #Algorithm #BinarySearch #Search #Python

## 1. 개념 및 목적

이진 탐색은 **오름차순으로 정렬된 리스트**에서 특정한 값의 위치를 찾는 알고리즘입니다. 탐색 범위를 반복적으로 절반씩 나누어 데이터가 위치할 가능성이 없는 절반을 버리는 방식으로 동작하며, 데이터의 양이 많을수록 선형 탐색($O(n)$)에 비해 훨씬 효율적입니다.

## 2. 상세 작동 원리

### 2.1. 탐색 과정

1. **정렬 확인:** 데이터가 오름차순(또는 내림차순)으로 정렬되어 있어야 합니다.
2. **변수 설정:** 탐색 범위의 시작 인덱스(`l`)와 끝 인덱스(`r`)를 설정합니다.
3. **중간값 비교:** 중간 위치의 값(`mid`)과 찾고자 하는 대상(`target`)을 비교합니다.
* `mid == target`: 탐색 성공, 해당 인덱스를 반환합니다.
* `mid > target`: `r`를 `mid - 1`로 옮겨 왼쪽 절반을 탐색합니다.
* `mid < target`: `l`를 `mid + 1`로 옮겨 오른쪽 절반을 탐색합니다.


4. **반복:** `l`가 `r`보다 커질 때까지 3번 과정을 반복합니다.

### 2.2. 시간 복잡도

한 번 비교할 때마다 탐색 범위가 1/2로 줄어들기 때문에 시간 복잡도는 **$O(\log n)$**입니다. 예를 들어, 데이터가 100만 개($2^{20} \approx 1,000,000$)일 때 단 20번의 비교만으로 원하는 값을 찾을 수 있습니다.

## 3. 예시 코드 (Python)

재귀 방식과 반복문 방식 중 실무에서 더 자주 쓰이는 반복문 방식을 예로 들어보겠습니다.

```python
def binary_search(arr, target):
    l = 0
    r = len(arr) - 1

    while l <= r:
        mid = (l + r) // 2
        
        # 찾고자 하는 값을 발견한 경우
        if arr[mid] == target:
            return mid
        # 중간값이 타겟보다 크면 왼쪽 절반 탐색
        elif arr[mid] > target:
            r = mid - 1
        # 중간값이 타겟보다 작으면 오른쪽 절반 탐색
        else:
            l = mid + 1
            
    return -1 # 값을 찾지 못한 경우

# 실행 결과 확인
my_list = [1, 3, 5, 7, 9, 11, 13, 15]
target_val = 7
result_idx = binary_search(my_list, target_val)

if result_idx != -1:
    print(f"원소 {target_val}는 인덱스 {result_idx}에 있습니다.")
else:
    print("원소를 찾을 수 없습니다.")

"""
예상 출력:
원소 7는 인덱스 3에 있습니다.
"""

```

## 4. 기술적 분석 (객관적 평가)

* **전제 조건:** 반드시 **정렬**이 되어 있어야 한다는 점이 가장 큰 제약입니다. 정렬되지 않은 데이터라면 정렬($O(n \log n)$) 비용까지 고려해야 합니다.
* **적용 범위:** 단순히 값을 찾는 것을 넘어, 특정 조건(예: $x$ 이상인 첫 번째 위치)을 만족하는 최솟값이나 최댓값을 찾는 **파라메트릭 서치(Parametric Search)**로 확장되어 최적화 문제 해결에 널리 쓰입니다.
* **주의 사항:** 중간 인덱스를 구할 때 `(l + r) // 2` 방식은 매우 큰 수의 덧셈에서 오버플로가 발생할 수 있습니다(Java 등 타 언어 기준). 파이썬은 정수 크기 제한이 없으나, 더 안전한 작성을 위해 `l + (r - l) // 2` 방식을 쓰기도 합니다.
