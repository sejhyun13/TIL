## 1. 개념 및 목적

람다 함수는 `def` 키워드를 사용하여 이름을 붙여 정의하는 일반 함수와 달리, **이름이 없는 익명 함수**입니다. 주로 함수를 인자로 받는 고차 함수(`map`, `filter`, `sort` 등) 내에서 일회성으로 사용할 간단한 로직을 정의하기 위해 사용합니다.

## 2. 상세 작동 원리

### 2.1. 기본 문법

`lambda 매개변수 : 표현식` 구조를 가집니다.

* **매개변수:** 함수에 전달할 입력값입니다. 여러 개일 경우 쉼표로 구분합니다.
* **표현식:** 함수가 실행할 로직이며, 계산된 결과값이 자동으로 반환(return)됩니다. `return` 키워드는 사용하지 않습니다.

```python
lambda x : x + 1

=

def func(x):
    return x + 1
```


### 2.2. 일반 함수와의 차이점

1. **익명성:** 함수 이름이 없으므로 변수에 할당하지 않는 한 재사용이 어렵습니다.
2. **간결함:** 복잡한 제어문(if-else 중첩, for문 등)을 포함할 수 없으며, 반드시 **단일 표현식**으로만 작성해야 합니다.
3. **메모리 효율:** 한 번 사용되고 사라지기 때문에 가비지 컬렉션(GC)에 유리하며 코드가 깔끔해집니다.

## 3. 예시 코드

실무와 알고리즘 문제 풀이에서 람다 함수가 가장 자주 쓰이는 패턴들입니다.

```python
# 1. 기본적인 사용법
add = lambda x, y: x + y
print(add(5, 3))  # 8

# 2. 리스트 정렬 시 키(key) 지정 (자주 쓰임)
pairs = [(1, 'one'), (3, 'three'), (2, 'two'), (4, 'four')]
# 튜플의 두 번째 요소(알파벳)를 기준으로 정렬
pairs.sort(key=lambda x: x[1])
print(pairs)
# 예상 출력: [(4, 'four'), (1, 'one'), (3, 'three'), (2, 'two')]

# 3. filter와 map에서의 활용
nums = [1, 2, 3, 4, 5, 6]
# 짝수만 걸러내기
evens = list(filter(lambda x: x % 2 == 0, nums))
# 각 숫자를 제곱하기
squares = list(map(lambda x: x**2, nums))

print(f"짝수들: {evens}")    # [2, 4, 6]
print(f"제곱값들: {squares}") # [1, 4, 9, 16, 25, 36]

```

## 4. 기술적 분석 (객관적 평가)

* **가독성:** 로직이 매우 짧을 때는 코드가 간결해져 가독성이 좋아지지만, 표현식이 길어지면 오히려 가독성을 해칠 수 있습니다. 이럴 때는 일반 `def` 함수를 선언하는 것이 좋습니다.
* **디버깅:** 이름이 없기 때문에 에러 발생 시 추적(Traceback)에서 함수 이름이 `<lambda>`로 표시되어 디버깅이 조금 더 까다로울 수 있습니다.
* **성능:** 일반 함수와 실행 속도 면에서는 큰 차이가 없으나, 함수 객체를 생성하는 오버헤드를 줄이는 목적으로 사용됩니다.
